# DESIGN.md —— 安防监控管理平台前端设计规范
## 适用场景
安防监控平台、设备运维平台、实时告警平台、视频预览与录像回放系统、智能接口治理后台  
技术栈：Element Plus / Vue 3 / Vite / TypeScript / Pinia / Vue Router

## 设计原则（当前项目）
- **监控优先**：界面首先服务于告警、视频、设备状态，重点信息必须一眼可见
- **信息密度适中**：保留 B 端高信息量特征，但避免传统后台过于生硬拥挤
- **浅色主应用 + 深色专项场景**：常规业务页使用浅色体系，登录页、监控页、驾驶舱使用深色强化沉浸感
- **统一反馈**：按钮、表单、状态标签、弹窗、列表、连接状态采用统一语义色与交互反馈
- **兼顾工程与视觉**：设计规范必须能够直接映射到当前 `tokens.css`、`components.css` 和页面样式实现

## 3. 间距与布局（当前实现）
### 基础单位
- 基础间距：8px
- 常规组件间距：12px / 16px
- 页面区块间距：18px / 24px
- 页面内边距：24px
- 卡片头部内边距：18px 22px
- 卡片主体内边距：20px 22px 22px

### 整体布局
- 布局结构：左侧菜单 + 顶部栏 + 主内容区
- 左侧菜单宽度：232px
- 折叠菜单宽度：84px-88px
- 顶部栏高度：60px
- 主内容区内边距：24px
- 页面背景：浅蓝灰渐变底

### 页面栅格
- 常规列表页：纵向堆叠式布局，区块间距 18px
- 搜索区：多列网格，默认 4 列
- 统计卡片区：4 列指标卡
- dashboard 图表区：2 列不等宽布局，主区约 1.4fr，辅区约 1fr
- 登录页：左右双栏，品牌区 + 表单区

### 响应式规则
- `1200px` 以下：搜索区和统计卡片降为 2 列
- `960px` 以下：主布局允许纵向堆叠，表单、统计、搜索降为 1 列
- `900px` 以下：登录页双栏切为单栏

## 5. 全局页面规范
### 页面背景
- 主应用背景使用浅色渐变，不使用纯白或纯灰硬底
- 背景通过左上角浅蓝色径向渐变提升层次

### 顶栏
- 顶栏固定吸顶
- 背景为半透明白色，带 `backdrop-filter: blur(14px)`
- 左侧为折叠按钮、标题、面包屑
- 右侧为连接状态、声音开关、用户信息、退出登录

### 侧边栏
- 整体深蓝灰背景
- 品牌区单独加深
- 一级分组使用按钮式折叠头
- 当前实现允许多个菜单组切换，但路由激活时会自动展开所属分组
- 菜单折叠后仅保留图标和 logo

### 主内容区
- 主内容固定使用 24px 内边距
- 列表页、详情页、统计页统一包裹在卡片或功能面板内
- 页面之间通过统一卡片、搜索区、指标区形成节奏

## 6. 核心组件规范（结合当前样式实现）
### 按钮
- 默认高度：40px
- 内边距：0 16px
- 字重：600
- 主按钮：蓝色渐变背景，白字，带投影
- 成功按钮：绿色渐变背景
- 警告按钮：橙色渐变背景
- 危险按钮：红色渐变背景
- 次按钮：白底或近白底，浅边框，轻阴影
- Hover：轻微上浮 `translateY(-1px)`，不使用夸张动画

### 表单
- 输入框默认高度：40px
- 文本域最小高度：88px
- 标签字号：13px，600
- 表单边框使用浅灰蓝边框
- Focus 态使用蓝色描边 + 4px 外发光
- 搜索表单背景采用浅渐变容器，而不是裸露表单项

### 表格
- 表格背景：纯白
- 表头背景：`--color-table-head`
- 单元格内边距：13px 14px
- 边框：浅灰蓝描边
- Hover 行背景：浅蓝高亮
- 紧凑列表页表格可降到 12px 字号和 9px 10px 内边距
- 操作列使用按钮组，不使用过于分散的文字链接

#### 日志/记录类表格落地规范
- 设备日志、巡检计划、执行记录等记录类页面应优先复用设备日志表格样式：`app-table` + `table-layout: fixed` + `colgroup` 固定列宽。
- 表格字号统一为 12px，单元格内边距使用 9px 10px，表头保持 12px 加粗，适合高频查看和较高信息密度列表。
- 主名称列可使用双行结构：第一行 13px 深色加粗，第二行 11px 辅助信息；辅助信息超长时使用省略号，不撑开表格。
- 时间、状态、数量、操作列必须显式设置宽度；操作列按钮使用紧凑按钮组，列宽不足时优先横向滚动，不允许挤压其它业务列。
- 空状态统一使用 `app-table__empty`，文案保持“暂无数据 / 暂无执行记录 / 加载中...”这类短文案。
- 分页统一放在表格右下角，布局使用 `total, prev, pager, next`；记录类页面即使 0 条也应显示 total，便于用户确认查询结果。

### 卡片
- 背景：白色
- 圆角：12px
- 边框：浅蓝灰描边
- 阴影：`--shadow-card`
- 头部与内容区分层明显，头部有底边线

### 状态标签
- 成功：绿色浅底 + 深绿文字
- 警告：橙色浅底 + 深橙文字
- 危险：红色浅底 + 深红文字
- 信息：蓝色浅底 + 深蓝文字
- 默认：灰底灰字

### 弹窗
- Dialog 圆角：18px
- 阴影：浮层级阴影
- Header 内边距：20px 24px 10px
- Body 内边距：12px 24px 24px
- 适合当前设备配置、详情展示、编辑表单场景

## 7. 业务场景专项规范
### 首页驾驶舱
- 背景采用深色仪表盘风格
- 指标卡使用深蓝面板、细边框和高亮数字
- 图表与指标强调“监控中”氛围，不使用浅色卡片风
- 适用于告警趋势、设备统计、推送统计等总览场景

### 登录页
- 登录页是品牌展示型页面，不沿用主应用浅色布局
- 整体使用深蓝渐变背景
- 中间为大面板，左侧品牌说明，右侧登录表单
- 登录卡片接近白色，形成强对比，保证输入效率

### 监控预览与录像回放
- 视频类页面应优先保证黑底或深底承载视频区域
- 工具栏和控制面板可使用深色半透明或深色渐变容器
- 功能优先级顺序：播放内容 > 设备选择 > 控制条 > 辅助信息
- 视频页面不建议套用普通 CRUD 卡片视觉

### 实时告警与设备列表
- 保持统一列表页结构：统计概览 + 搜索区 + 表格区
- 高等级告警使用明确语义色，不使用模糊中性态
- 实时连接状态必须在头部长期可见

### 智能接口与推送配置
- 这类页面信息复杂，适合使用卡片分组、标签页、详情抽屉
- 原始 JSON、标准事件载荷、日志信息推荐使用深浅对比明确的代码块或说明面板

## 8. 菜单规范（按当前实现）
### 左侧菜单基础规则
- 背景主色：`#30455f` 到 `#2f4255` 深蓝灰系
- 品牌区高度：约 74px-86px
- 一级菜单高度：约 50px
- 二级菜单高度：约 46px
- 一级、二级文字统一使用 15px 左右，不走小字号弱化路线

### 一级菜单
- 左侧图标 + 文字 + 右侧折叠箭头
- 激活态使用蓝色系渐变高亮和轻投影
- Hover 态仅做轻量背景变化

### 二级菜单
- 左内边距明显大于一级菜单，形成层级感
- 激活态允许蓝色渐变背景或高亮文字
- 当前项目实现中，激活态会强化文字颜色并增加阴影/高亮底

### 折叠菜单
- 折叠后隐藏文字和子项
- 品牌 logo 缩小
- 当前实现未为折叠态补充完整悬浮提示规范，后续可补强 tooltip 体验
- 菜单结构：必须使用风琴式菜单，同一时刻仅展开一个一级分组；进入某个页面时，所属分组自动展开

## 9. 状态与反馈规范
### 实时连接状态
- 状态展示位置固定在顶栏右侧
- 通过圆点颜色区分：连接成功绿色，连接中/重连中橙色，默认灰色
- 状态文案必须明确，如“已连接”“连接中”“重连中”

### 声音告警开关
- 在顶栏显式显示开关状态
- 开启/关闭必须一眼可辨，不隐藏在二级菜单中

### 通知反馈
- 高等级实时告警使用全局通知弹出
- 业务操作采用 Element Plus 消息与弹窗反馈
- 错误态优先使用清晰文案，不只给出失败码

### 表单与操作反馈
- Focus、Hover、Disabled、Danger 必须视觉一致
- 删除、停用、断开类操作一律走危险色

## 10. 当前项目设计风格总结
- 风格关键词：`科技蓝`、`轻拟物卡片`、`浅色管理后台`、`深色监控场景`
- 与标准纯后台相比，当前项目更强调监控与视频场景的沉浸感
- 与大屏系统相比，当前项目仍保留 B 端管理效率，不走全视觉化路线
- 与 `DESIGN_Example.md` 相比，当前项目真实实现的按钮、菜单和卡片更具渐变和阴影表现

---

## 11. 禁止项（基于当前项目约束）
- ❌ 不要把主应用整体改成纯黑大屏风格，当前系统仍是管理后台主导
- ❌ 不要在普通列表页过度使用深色面板，深色只用于登录、监控、驾驶舱等专项页面
- ❌ 不要混用多套主色，当前必须以蓝色体系为主
- ❌ 不要引入夸张动画、长链路过渡和过强毛玻璃
- ❌ 不要让表格、搜索区、卡片出现完全不同的圆角和阴影体系
- ❌ 不要在无权限场景下只做弱提示，当前权限策略偏向直接隐藏

---

## 12. 落地建议
- 继续以 `frontend/src/styles/tokens.css` 作为视觉 token 单一来源
- 新页面优先复用 `page-card`、`search-form`、`status-tag`、`app-button`、`app-table` 等通用样式
- 视频、监控、回放类页面允许专项样式，但需保持主色、文字和状态反馈一致
- 后续如果补齐占位页面，应优先沿用当前浅色业务页规范，不另起一套设计语言
