# 测试与验收文档

本文档用于指导开发、联调、发布前的验证工作。当前项目自动化测试较少，因此建议结合“命令级检查 + 接口验证 + 页面验收 + 设备联调”完成质量把关。

## 1. 测试分层

| 层级 | 目标 | 推荐方式 |
| --- | --- | --- |
| 单元测试 | 验证工具函数、纯业务逻辑 | `go test ./...` |
| 构建测试 | 验证 Go、TypeScript、Vite 构建可用 | `go test`、`npm run build` |
| 接口测试 | 验证 API 入参、鉴权、响应格式 | curl、Postman、Apifox |
| 页面测试 | 验证登录、菜单、表格、弹窗、权限 | 浏览器手工验收 |
| 集成测试 | 验证 MySQL、SDK、视频、SSE、推送 | 本地或测试环境联调 |
| 发布验收 | 验证部署环境完整可用 | 部署清单 + 核心流程回归 |

## 2. 后端基础检查

在 `secmgmt_go` 目录执行：

```powershell
go test ./...
```

如果 Windows 环境下 Go build cache 无权限，可临时指定项目内缓存：

```powershell
$env:GOCACHE = Join-Path (Get-Location) ".gocache"
go test ./...
```

检查重点：

- 所有包能编译通过。
- `internal/util` 等已有测试通过。
- 没有因为 build tags、CGO、SDK 路径导致编译失败。

## 3. 前端构建检查

在 `secmgmt_go/frontend` 目录执行：

```powershell
npm run build
```

如果 npm 缓存目录无权限，可临时指定项目内缓存：

```powershell
$env:npm_config_cache = Join-Path (Get-Location) ".npm-cache"
npm run build
```

检查重点：

- `vue-tsc` 类型检查通过。
- Vite 构建通过。
- 构建产物可被 Nginx 正确托管。
- 大 chunk 警告不是构建失败，但建议后续做按需加载或分包优化。

## 4. 数据库初始化验收

初始化脚本：

```text
sql/init_database.sql
```

检查项：

- 数据库字符集为 `utf8mb4`。
- 初始化脚本可重复在空库执行成功。
- 用户、角色、菜单、权限等基础数据存在。
- 设备、告警、推送、智能接口相关表存在。
- 时间字段、状态字段、索引符合业务查询需要。

建议 SQL 检查：

```sql
SHOW TABLES;
SELECT COUNT(*) FROM sys_users;
SELECT COUNT(*) FROM sys_roles;
SELECT COUNT(*) FROM sys_menus;
SELECT COUNT(*) FROM sys_permissions;
```

## 5. API 基础验收

健康检查：

```powershell
curl http://127.0.0.1:8000/healthz
```

登录：

```powershell
curl -X POST http://127.0.0.1:8000/api/auth/login `
  -H "Content-Type: application/json" `
  -d "{\"username\":\"admin\",\"password\":\"<password>\"}"
```

带 token 请求：

```powershell
curl http://127.0.0.1:8000/api/auth/me `
  -H "Authorization: Bearer <token>"
```

通用接口验收点：

- 未登录访问保护接口应返回 401。
- 登录后访问有权限页面数据应成功。
- 请求参数错误应返回明确错误信息。
- 写操作应正确落库。
- 删除、禁用后列表状态应同步变化。

## 6. 登录与权限验收

验收流程：

1. 使用管理员账号登录。
2. 检查首页、设备、告警、推送、系统管理等菜单是否正常显示。
3. 新建一个普通角色，仅分配部分菜单。
4. 新建普通用户并绑定该角色。
5. 使用普通用户登录。
6. 检查未授权菜单不显示。
7. 直接访问未授权路由时，应看到无权限状态或被拦截。

重要提醒：

- 当前后端主要做 JWT 登录态校验。
- 接口级权限校验仍需加强，测试时应记录“前端不可见但直接调接口是否可用”的结果。

## 7. 设备管理验收

摄像机：

- 新增摄像机。
- 编辑摄像机。
- 启停摄像机。
- 测试连接。
- 检测状态。
- 查看 SDK 配置。
- 修改网络、图像、录像、PTZ 等配置。

录像机：

- 新增录像机。
- 测试连接。
- 同步通道。
- 查看通道列表。
- 编辑通道。

验收注意：

- 设备 IP、端口、账号密码必须真实可达。
- Windows 与 Linux SDK 行为可能不同，应在目标部署系统验证。
- 设备密码加密依赖 `DEVICE_SECRET_KEY`，更换密钥后需验证历史数据可用性。

## 8. 视频能力验收

实时预览：

1. 进入监控预览页面。
2. 选择摄像机或通道。
3. 检查画面是否播放。
4. 停止播放后确认后端资源释放。

录像回放：

1. 进入录像查看页面。
2. 选择设备、时间范围。
3. 搜索录像片段。
4. 获取播放地址。
5. 拖动进度。
6. 下载录像文件。

常见验收点：

- 播放地址是否可访问。
- 浏览器控制台是否有跨域、MIME、资源路径错误。
- Nginx 是否正确托管 `public/codebase` 下静态资源。
- 录像下载文件名是否正确。

## 9. 告警与 SSE 验收

实时告警：

- 打开实时告警页面。
- 建立 SSE 连接。
- 触发或模拟告警。
- 页面应出现实时通知。

告警处理：

- 查询告警列表。
- 查看告警详情。
- 执行处理。
- 标记误报。
- 重新推送。
- 检查处理日志和状态变化。

SSE 检查：

- token 过期后连接应失败或触发重新登录。
- 网络断开后前端应能重连或提示。
- 同一告警不应重复弹出过多通知。

## 10. 推送模块验收

流程：

1. 新增推送配置。
2. 测试推送配置。
3. 触发告警推送。
4. 查看推送日志。
5. 对失败日志执行重试。

检查项：

- 请求超时是否受 `PUSH_HTTP_TIMEOUT_SECONDS` 控制。
- 失败原因是否记录清楚。
- 密钥、Webhook、Header 等敏感配置不应明文暴露给前端。

## 11. 智能接口与 AI 回调验收

智能接口：

- 新增厂商。
- 配置能力。
- 绑定设备。
- 配置规则。
- 调用事件接入接口。
- 检查原始事件和标准事件。

AI 复核：

- 对智能事件提交 AI 复核。
- 查询 AI 任务。
- 模拟 AI 回调。
- 查看复核结果。
- 失败任务执行重试。

注意：

- 当前部分接入和回调接口在 JWT 保护分组下。
- 如果第三方系统无法携带 JWT，需要先调整后端路由和验签策略，再做联调。
